# Module 1 – Microsoft Sentinel Overview & Lab Setup

## Overview

This introductory module focused on understanding the Microsoft Sentinel architecture and preparing the lab environment for future exercises.

The Microsoft Sentinel Training Lab solution was deployed into an Azure Log Analytics Workspace, providing sample telemetry that would be used throughout the remaining modules.

---

## Objectives

- Understand Microsoft Sentinel architecture
- Learn the SIEM/SOAR workflow
- Explore Microsoft Defender Portal
- Prepare the Log Analytics Workspace
- Deploy the Microsoft Sentinel Training Lab solution

---

## Technologies Used

- Microsoft Sentinel
- Microsoft Defender Portal
- Microsoft Azure
- Azure Log Analytics Workspace

---

## Key Concepts Learned

- SIEM vs SOAR
- Analytics Rules
- Alerts
- Incidents
- Data Connectors
- Log Analytics Workspace
- Kusto Query Language (KQL)
- Microsoft Defender integration

---

## Learning Outcomes

After completing this module I was able to:

- Navigate the Microsoft Defender portal
- Understand how Microsoft Sentinel processes security data
- Recognize the relationship between logs, analytics rules, alerts, and incidents
- Prepare the lab environment for subsequent hands-on exercises

---

## Notes

This module focused primarily on environment preparation and foundational concepts. The practical configuration and evidence collection begin in **Module 2**, where Scheduled Analytics Rules are created and deployed.
